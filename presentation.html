Problem Statement
THE PROBLEM
Banking Data is Highly Sensitive
Traditional RAG systems store raw document embeddings in vector databases. Account numbers, IFSC codes, customer PII, and financial details can be leaked, reconstructed, or misused by adversaries who gain access to these vectors.
KEY THREATS
Identified Attack Vectors
  • → Embedding Inversion Attacks
  • → Prompt Injection / LLM Jailbreaks
  • → SQL & Command Injection via PDF
  • → PII Data Leakage via RAG Retrieval
  • → Cross-user Data Access (Tenant Isolation)
OUR SOLUTION
Secure RAG with 5-Layer Defence Pipeline
A purpose-built secure pipeline that sanitizes, pseudonymizes, and protects every document before it enters the vector database — while preserving full retrieval accuracy and role-based access control.

Embedding Inversion Attack & Defence
The Attack Scenario
Without Protection (Vulnerable)
Attacker steals Pinecone API key → dumps all vectors → uses ML model to reconstruct text from embeddings.

Cosine similarity: 1.000000 (perfect match)
Attack success rate: ~100%
With Our DP Protection (Secure)
Gaussian noise (σ=0.01) added to every stored embedding. Re-normalized to unit sphere — search still works accurately.

Inversion confidence degradation: 12.3%
Attack success rate: ~0%

3-Layer Defence Stack
Layer 1: PII Pseudonymization (real data never in embedding)
Layer 2: Gaussian DP Noise σ=0.01 (inversion fails)
Layer 3: Pinecone owner_id filter (no cross-user access)
Role-Based Access Control (RBAC)
👑
ADMIN ROLE
Global Knowledge Base
  • Upload banking policy PDFs
  • Documents shared with all users
  • Stored with owner_id = "admin"
  • Pinecone namespace: global
👤
USER ROLE
Private Knowledge Base
  • Upload personal documents only
  • Isolated to own namespace
  • Stored with owner_id = username
  • Cannot access other users' data
Query Isolation Logic
# User queries always include admin docs
owner_ids = ["admin"]
if user["role"] != "admin":
    owner_ids.append(user["sub"])

# Pinecone filters by owner_id server-side
results = index.query(
    filter={"owner_id": {"$in": owner_ids}}
)
Security Guarantee
User A cannot access User B's documents even if they query about the same topic. The Pinecone filter is enforced server-side — not client trust-based.
